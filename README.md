# Канапе — сайт кейтеринга

Статический сайт-меню для кейтеринговой компании. Хостится на GitHub Pages.

## Структура проекта

```
├── index.html              # Точка входа (пустые контейнеры, всё рендерит JS)
├── css/
│   └── style.css           # Все стили + responsive breakpoints
├── js/
│   └── app.js              # Рендеринг всех секций из JSON
├── data/
│   └── menu.json           # Все данные сайта (контент, меню, контакты)
├── assets/
│   └── images/             # Изображения
│       ├── logo.webp
│       ├── hero.webp
│       ├── about-chef.webp
│       ├── products/       # Фото блюд
│       └── events/         # Фото мероприятий
├── package.json
└── .github/
    └── workflows/
        └── deploy.yml      # Автодеплой на GitHub Pages
```

## Как добавить изображения

Все пути к изображениям задаются в [`data/menu.json`](data/menu.json). Формат — относительный путь от корня сайта:

```json
"image": "assets/images/products/caprese.webp"
```

Рекомендуемые форматы: **WebP** (основной), JPG/PNG (запасной).

### Размеры изображений

| Место | Рекомендуемый размер |
|-------|---------------------|
| Логотип | 200×80 px |
| Hero | 1200×800 px |
| About (шеф) | 800×600 px |
| Продукты | 400×300 px |
| Мероприятия | 600×400 px |

## Как обновить контент

Весь контент хранится в [`data/menu.json`](data/menu.json):

- **header** — логотип, навигация, контакты, кнопка CTA
- **hero** — заголовок, текст, изображение
- **menu** — категории и позиции меню с описаниями
- **about** — текст о компании, статистика, изображение
- **events** — галерея мероприятий с описаниями
- **footer** — соцсети, навигация, контакты, реквизиты

## Деплой на GitHub Pages

### Первый раз

1. Создайте репозиторий на GitHub (публичный или приватный с Pages)
2. Инициализируйте git и сделайте первый коммит:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПО.git
git push -u origin main
```

3. В настройках репозитория: **Settings → Pages → Source → GitHub Actions**

После пуша в `main` сайт автоматически задеплоится. Адрес:
```
https://ВАШ_ЛОГИН.github.io/ВАШ_РЕПО/
```

### Обновление сайта

```bash
git add .
git commit -m "Обновление меню"
git push
```

Деплой занимает ~1 минуту. Статус можно отслеживать во вкладке **Actions** репозитория.

## Локальный запуск

```bash
npm install
npm run dev
```

Сайт откроется на `http://localhost:3000`.

> **Важно:** открывать через сервер, а не напрямую как файл — иначе `fetch('data/menu.json')` не сработает из-за CORS.

## Технологии

- Чистый HTML/CSS/JS, без фреймворков
- Данные загружаются из JSON через `fetch`
- Responsive: 1440px → 1180px → 900px → 480px → 360px
- GitHub Actions для автодеплоя
